4. Merge, rebase i cherry-pick

Merge

git merge feature

Tworzy połączenie dwóch historii. Zachowuje strukturę rozgałęzienia.

Rebase

git rebase main

Przenosi commity na nową podstawę i tworzy liniową historię.

Cherry-pick

git cherry-pick C3

Kopiuje pojedynczy commit do aktualnej gałęzi.

Wpływ na pliki

5. Cofanie zmian

restore

git restore plik

Odrzuca niezacommitowane zmiany w pliku.

reset

git reset --soft HEAD~1

Przesuwa wskaźnik gałęzi. Wariant określa, co stanie się ze staging i plikami.

revert

git revert C

Tworzy nowy commit odwracający wcześniejszą zmianę.

Dodatkowe polecenia: touch NAZWA, edit NAZWA, git status, git diff, git add ., git commit -m "opis", git log --oneline --graph, git switch -c NAZWA, git switch NAZWA, git merge NAZWA, git rebase NAZWA, git cherry-pick ID, git revert HEAD, git reset --soft HEAD~1, git reset --hard HEAD~1, git stash, git stash pop, git fetch, git pull, git push.
$
Polecenia działają na tym samym stanie co przyciski. Błędne sekwencje generują komunikaty diagnostyczne.
