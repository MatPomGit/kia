1. Cele i rezultaty uczenia się

Znaczenie praktyczne. Skrypt powłoki jest zapisanym programem automatyzującym sekwencję poleceń. Pozwala odtworzyć proces w identyczny sposób, ogranicza liczbę błędów ręcznych i stanowi podstawę wielu zadań administracyjnych, testowych oraz wdrożeniowych.

Tworzenie skryptów

Budowa pliku wykonywalnego, shebang, uprawnienia i uruchamianie.

Sterowanie przepływem

Instrukcje if, case, for i while.

Walidacja danych

Sprawdzanie liczby argumentów, istnienia plików i poprawności wejścia.

Edycja tekstowa

Podstawowa praca w VIM: poruszanie, wyszukiwanie, edycja i zapis.

2. Tworzenie pierwszego skryptu

Skrypt jako plik programu. Bash czyta plik linia po linii i wykonuje zawarte w nim polecenia. Pierwsza linia wskazuje interpreter, który ma przetworzyć plik.

2.1. Utworzenie katalogu i pliku

mkdir -p "$HOME/cw3"
cd "$HOME/cw3"
cat > pkt1 <<'EOF'
#!/bin/bash
echo "Test polecenia skryptu"
EOF

2.2. Dlaczego samo pkt1 nie działa?

Powłoka szuka programów w katalogach zapisanych w zmiennej PATH. Bieżący katalog zwykle nie jest tam uwzględniony.

pkt1
./pkt1
Zapis ./pkt1 oznacza: uruchom plik pkt1 znajdujący się w bieżącym katalogu.

2.3. Nadanie uprawnień

chmod 744 pkt1
./pkt1

Model uprawnień. Wartość 744 oznacza: właściciel może czytać, zapisywać i wykonywać plik, natomiast pozostali użytkownicy mogą go tylko czytać. W praktyce skrypt prywatny często otrzymuje uprawnienia 700 lub 744.

3. Instrukcje warunkowe

Decyzje w skrypcie. Instrukcja if pozwala wykonać różne operacje zależnie od wyniku testu. W Bash sukces testu jest reprezentowany kodem zakończenia 0.

3.1. Sprawdzanie istnienia pliku

cat > pkt2 <<'EOF'
#!/bin/bash

if [ -f pkt1 ]; then
  echo "Plik pkt1 istnieje."
fi

if [ ! -f dane1 ]; then
  echo "Linia tekstu w danych1" > dane1
fi
EOF

chmod 744 pkt2
./pkt2
TestZnaczenie
-f plikIstnieje zwykły plik
-d katalogIstnieje katalog
-e obiektIstnieje dowolny obiekt systemu plików
! warunekNegacja warunku

4. Zmienne i dane wejściowe

Zmienne w Bash. Zmienne nie wymagają deklaracji typu. Domyślnie przechowują tekst, dlatego porównania liczbowe i tekstowe korzystają z różnych operatorów.

4.1. Porównanie dwóch liczb

cat > pkt3 <<'EOF'
#!/bin/bash

read -r -p "Podaj pierwszą liczbę: " a
read -r -p "Podaj drugą liczbę: " b

if [ -z "$a" ] || [ -z "$b" ]; then
  echo "Błąd: należy podać dwie wartości." >&2
  exit 1
fi

if [ "$a" -gt "$b" ]; then
  echo "$a jest większe od $b"
elif [ "$a" -lt "$b" ]; then
  echo "$b jest większe od $a"
else
  echo "Wartości są równe."
fi
EOF

chmod 744 pkt3
./pkt3
OperatorZnaczenie
-gtwiększe niż
-ltmniejsze niż
-eqrówne
-neróżne
-złańcuch pusty
Przekierowanie >&2 wysyła komunikat do standardowego strumienia błędów. Dzięki temu komunikaty diagnostyczne można oddzielić od zwykłego wyniku programu.

5. Polecenia iteracyjne

Automatyzacja powtarzalnych operacji. Pętla wykonuje blok poleceń wielokrotnie. for iteruje po zbiorze elementów, a while działa dopóki warunek pozostaje prawdziwy.

5.1. Rozpoznawanie typów obiektów

cat > pkt4 <<'EOF'
#!/bin/bash

for obiekt in *
do
  if [ -d "$obiekt" ]; then
    echo "$obiekt — katalog"
  elif [ -f "$obiekt" ]; then
    echo "$obiekt — plik"
  elif [ -e "$obiekt" ]; then
    echo "$obiekt — inny obiekt"
  fi
done
EOF

chmod 744 pkt4
./pkt4

5.2. Wielokrotne wyświetlenie tekstu

cat > pkt5 <<'EOF'
#!/bin/bash

read -r -p "Podaj tekst: " tekst
read -r -p "Podaj liczbę powtórzeń: " n

i=1
while [ "$i" -le "$n" ]
do
  echo "$tekst"
  i=$((i + 1))
done
EOF

chmod 744 pkt5
./pkt5
Każda pętla while musi zawierać mechanizm zmiany warunku. Brak aktualizacji zmiennej sterującej prowadzi do pętli nieskończonej.

6. Instrukcja case i pętla sterująca

Wielowariantowy wybór. case jest czytelniejsza od wielu zagnieżdżonych instrukcji if, gdy program reaguje na skończony zestaw poleceń użytkownika.

cat > pkt6 <<'EOF'
#!/bin/bash

while true
do
  read -r -p "Polecenie [a/b/c/x/liczba]: " wybor

  case "$wybor" in
    a)
      : > daty.log
      echo "Utworzono pusty plik daty.log"
      ;;
    b)
      date >> daty.log
      echo "Dodano datę."
      ;;
    c)
      if [ -f daty.log ]; then
        cat daty.log
      else
        echo "Plik daty.log nie istnieje."
      fi
      ;;
    x)
      echo "Koniec programu."
      break
      ;;
    ''|*[!0-9]*)
      echo "Nieprawidłowe polecenie."
      ;;
    *)
      i=1
      while [ "$i" -le "$wybor" ]
      do
        date
        i=$((i + 1))
      done
      ;;
  esac
done
EOF

chmod 744 pkt6
./pkt6

Wzorce w case. Wariant ''|*[!0-9]* przechwytuje pusty tekst lub wartość zawierającą znaki inne niż cyfry. Pozostały wariant może być traktowany jako liczba dodatnia.

7.1. Ćwiczenie

  1. Otwórz skrypt: vi pkt3.
  2. Przejdź do początku pliku poleceniem gg.
  3. Wyszukaj read przez /read.
  4. Dodaj przed pierwszym read ustawienie IFS=,.
  5. Zapisz plik i uruchom go wewnątrz VIM: :!./pkt3.
  6. Podaj wartości oddzielone przecinkiem i przeanalizuj wynik.

8. Zadania dodatkowe

8.1. Monitorowanie CPU i pamięci podręcznej w macOS

Cel zadania. Skrypt ma cyklicznie pobierać dane z narzędzi systemowych, przetwarzać je i reagować na sygnał przerwania. Jest to model typowy dla prostych agentów monitorujących.

#!/bin/bash

trap 'echo; echo "Monitorowanie zakończono: $(date)"; exit 0' INT

while true
do
  cpu=$(top -l 1 | awk '/CPU usage/ {print 100 - $7}')

  file_backed=$(vm_stat | awk '/File-backed pages/ {gsub("\\.","",$3); print $3}')
  purgeable=$(vm_stat | awk '/Pages purgeable/ {gsub("\\.","",$3); print $3}')

  cached_gb=$(awk -v a="$file_backed" -v b="$purgeable" \
    'BEGIN {printf "%.2f", ((a+b)*4096)/(1024^3)}')

  printf "%s | CPU: %s%% | Cache: %s GB\n" \
    "$(date '+%Y-%m-%d %H:%M:%S')" "$cpu" "$cached_gb"

  sleep 1
done

8.2. Symulacja zajmowania przestrzeni dyskowej

Uwaga terminologiczna. Tworzenie plików nie jest równoważne bezpośredniej alokacji pamięci RAM. Zadanie symuluje zajmowanie zasobu przez pliki tymczasowe, a wpływ na pamięć zależy od mechanizmu buforowania systemu operacyjnego.

#!/bin/bash

block_size=100
target_mb="${1:-0}"
tmp_dir=$(mktemp -d)

cleanup() {
  echo
  echo "Usuwanie plików tymczasowych..."
  rm -rf "$tmp_dir"
  echo "Zwolniono zasoby."
  exit 0
}

trap cleanup INT TERM

if ! [[ "$target_mb" =~ ^[0-9]+$ ]] || [ "$target_mb" -le 0 ]; then
  echo "Użycie: $0 LICZBA_MB" >&2
  exit 1
fi

allocated=0
index=1

while [ "$allocated" -lt "$target_mb" ]
do
  remaining=$((target_mb - allocated))

  if [ "$remaining" -lt "$block_size" ]; then
    current="$remaining"
  else
    current="$block_size"
  fi

  mkfile -n "${current}m" "$tmp_dir/blok_${index}.bin"
  allocated=$((allocated + current))

  echo "Przydzielono: ${allocated}/${target_mb} MB"
  index=$((index + 1))
done

echo "Zakończ program przez Ctrl+C, aby usunąć pliki."
while true
do
  sleep 1
done

10. Zakończenie ćwiczenia

Higiena środowiska. Usunięcie katalogu ćwiczeniowego zapobiega pozostawieniu danych i zapewnia powtarzalny stan stanowiska dla kolejnej grupy.

rm -rf "$HOME/cw3"
Przed wykonaniem polecenia sprawdź ścieżkę. Nie używaj rm -rf bez jednoznacznie wskazanego katalogu.

Na końcu zakończ sesję poleceniem:

exit
