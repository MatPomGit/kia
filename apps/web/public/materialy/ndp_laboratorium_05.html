Laboratorium 5
Pakiety LaTeX i własne makra

Rozszerzanie dokumentu LaTeX za pomocą pakietów, modyfikacja numeracji i podpisów, definiowanie makr, konfiguracja list i listingów, kolory, metadane PDF, załączniki, bibliografia BibTeX oraz rysowanie obwodów w circuitikz.

czas: 2 × 45 min tryb: praca indywidualna narzędzia: VS Code + LaTeX + BibTeX poziom: średni
Spis treści 1. Cele 2. Przygotowanie 3. Polskie standardy składu 4. Własne makra 5. Listy i środowiska 6. Matematyka 7. Kolory 8. Listingi kodu 9. Metadane i załączniki 10. Bibliografia 11. Circuitikz 12. Lista kontrolna 13. Zakończenie 14. Dodatkowe środowiska

1. Cele i rezultaty uczenia się

Znaczenie praktyczne. Pakiety rozszerzają możliwości LaTeX bez konieczności samodzielnego implementowania całych mechanizmów. Makra pozwalają natomiast ujednolicić skład i ograniczyć powtarzanie kodu źródłowego.

Konfiguracja dokumentu

Dostosowanie podpisów, numeracji i elementów typograficznych do przyjętego standardu.

Reużywalność

Definiowanie własnych poleceń i globalnych stylów.

Treści techniczne

Wzory, listingi kodu, kolory, bibliografia i załączniki.

Dokument końcowy

Metadane PDF, osadzone pliki i elementy grafiki wektorowej.

2. Przygotowanie dokumentu

Kontrola punktu startowego. Przed wprowadzeniem zmian należy skompilować dokument bazowy. Dzięki temu wiadomo, czy późniejsze błędy wynikają z nowych modyfikacji, czy z niepoprawnego środowiska lub brakujących plików.

  1. Pobierz i rozpakuj archiwum startowe.
  2. Otwórz plik wprowadzenie.tex.
  3. Sprawdź obecność plików pomocniczych, np. fragmenty.tex, zrodlo.c i zbior.bib.
  4. Skompiluj dokument przed rozpoczęciem pracy.
pdflatex wprowadzenie.tex
Nie kopiuj kodu LaTeX bezpośrednio z pliku PDF, jeżeli dostępny jest plik tekstowy z listingami. PDF może zawierać typograficzne cudzysłowy, ligatury lub niewidoczne znaki.

3. Dostosowanie do krajowych standardów składu

Po co zmieniać podpisy? W publikacjach technicznych stosuje się określone konwencje skrótów, kropek i położenia podpisów. Pakiet caption pozwala sterować tymi elementami globalnie i spójnie.

3.1. Podpisy rysunków

\usepackage{caption}

\DeclareCaptionLabelFormat{cfscr}{#1 #2}
\DeclareCaptionFont{ftscr}{\fontsize{10}{12pt}\selectfont}
\DeclareCaptionLabelSeparator{labscr}{~}

\captionsetup[figure]{
  name={Rys. },
  labelformat=cfscr,
  labelfont=ftscr,
  labelseparator=labscr,
  textfont=ftscr,
  position=bottom,
  skip=3pt
}

\makeatletter
\renewcommand\thechapter{\@arabic\c@chapter.}
\renewcommand\thefigure{%
  \ifnum\c@chapter>\z@\thechapter\fi
  \@arabic\c@figure.
}
\makeatother

3.2. Podpisy tabel

\captionsetup[table]{
  name={Tab. },
  labelformat=cfscr,
  labelfont=ftscr,
  labelseparator=labscr,
  textfont=ftscr,
  position=top,
  skip=3pt
}

\makeatletter
\renewcommand\thetable{%
  \ifnum\c@chapter>\z@\thechapter\fi
  \@arabic\c@table.
}
\makeatother
Polecenia zawierające znak @ wymagają otoczenia przez \makeatletter i \makeatother. Znak ten jest zwykle zarezerwowany dla poleceń wewnętrznych LaTeX.

4. Definiowanie własnych makr

Makro jako abstrakcja. Własne polecenie ukrywa szczegóły formatowania. Jeżeli styl zmieni się później, wystarczy poprawić jedną definicję zamiast wszystkich wystąpień w dokumencie.

4.1. Numeracja sekcji, podsekcji i równań

\makeatletter
\renewcommand\thesection{\thechapter\@arabic\c@section.}
\renewcommand\thesubsection{\thesection\@arabic\c@subsection.}
\renewcommand\theequation{\thechapter\@arabic\c@equation}
\makeatother

4.2. Własne polecenie \kod

\newcommand{\kod}[1]{\texttt{#1}}
W~języku C funkcja \kod{main} oznacza miejsce,
od którego rozpocznie się wykonywanie programu.
Wewnątrz niej znajduje się deklaracja zmiennej
\kod{i} typu \kod{int}.
Parametr #1 oznacza pierwszy argument makra. Wywołanie \kod{main} przekazuje tekst main do definicji polecenia.

5. Pakiet enumitem i konfiguracja list

Globalne i lokalne style. Pakiet enumitem umożliwia osobne ustawienia dla całego dokumentu oraz dla pojedynczej listy. To lepsze rozwiązanie niż ręczne modyfikowanie każdej pozycji.

5.1. Ustawienia globalne

\usepackage{enumitem}

\setlist[itemize]{label=---,nosep}
\setlist[enumerate]{nosep}

5.2. Lista z lokalnym stylem

\begin{itemize}[
  label=*,
  leftmargin=2.25\parindent
]
  \item Pierwszy element
  \item Drugi element
  \item Trzeci element
\end{itemize}
Parametr leftmargin nie służy do tworzenia kolejnych poziomów zagnieżdżenia. Dla poziomów list używa się osobnych konfiguracji, np. \setlist[itemize,2]{...}.

6. Pakiet amsmath i wzory

Rozszerzony skład matematyczny. Pakiet amsmath zapewnia środowiska do macierzy, układów równań, wyrównywania wzorów i poprawnych odwołań.

\usepackage{amsmath}

6.1. Układ równań

\begin{equation}
\left\{
\begin{matrix}
y = 2x - 2 \\
x = -3y + 2
\end{matrix}
\right.
\label{eq:uklad}
\end{equation}

6.2. Odwołanie do równania

Układ przedstawiono w równaniu~\eqref{eq:uklad}.
\eqref automatycznie dodaje nawiasy wokół numeru równania, dlatego jest lepsze od ręcznego zapisu (\ref{...}).

7. Definiowanie i używanie kolorów

Kolor jako element stylu. Nazwany kolor można zdefiniować raz i używać w wielu miejscach. Ułatwia to zachowanie zgodności z identyfikacją wizualną instytucji.

\usepackage{xcolor}

\DefineNamedColor{named}{wydzialowy}{rgb}{
  0.2941,0.6274,0.647
}
\textcolor{wydzialowy}{
  Tekst w kolorze wydziałowym.
}

\textcolor{red}{
  Tekst w kolorze czerwonym.
}
Kolor nie powinien być jedynym nośnikiem znaczenia. W dokumentach dostępnych cyfrowo należy uwzględnić osoby z zaburzeniami rozpoznawania barw oraz wydruk czarno-biały.

8. Listingi kodu źródłowego

Kod jako treść techniczna. Pakiet listingów zachowuje wcięcia, numeruje wiersze i wyróżnia elementy składni. Jest przeznaczony do prezentacji kodu, a nie do jego wykonywania.

8.1. Dołączenie pakietu

\usepackage[final]{listingsutf8}

8.2. Globalna konfiguracja

\lstset{
  language={[ANSI]C},
  showspaces=false,
  showtabs=false,
  frame=single,
  numbers=left,
  stepnumber=1,
  extendedchars=true,
  basicstyle=\ttfamily\small,
  keywordstyle=\fontfamily{lmtt}\fontseries{b}
    \selectfont\small\color{blue},
  stringstyle=\color{olive},
  commentstyle=\color{green}\itshape,
  showstringspaces=false
}

8.3. Wczytanie pliku

\lstinputlisting[
  language={[ANSI]C},
  inputencoding=utf8/cp1250
]{zrodlo.c}
Parametr kodowania powinien odpowiadać rzeczywistemu kodowaniu pliku. Współcześnie preferuje się UTF-8, jednak starsze środowiska mogą wymagać konwersji.

9. Metadane PDF i osadzanie załączników

PDF jako nośnik informacji. Dokument PDF może zawierać nie tylko widoczną treść, ale również metadane, hiperłącza i pliki źródłowe dołączone jako załączniki.

9.1. Metadane i hiperłącza

\usepackage{hyperref}

\hypersetup{
  pdftitle={Pakiety LaTeX i własne makra},
  pdfsubject={Laboratorium z LaTeX},
  pdfauthor={Imię i nazwisko},
  pdfkeywords={LaTeX, pakiety, makra, BibTeX}
}

9.2. Załącznik w pliku PDF

\usepackage{embedfile}

\embedfile{zrodlo.c}
Nie wszystkie przeglądarki PDF wyświetlają osadzone pliki. Funkcję załączników najlepiej testować w pełnym czytniku PDF, np. Adobe Acrobat Reader.

10. Bibliografia z BibTeX

Rozdzielenie treści i danych bibliograficznych. Cytowanie odbywa się przez klucz, natomiast pełne dane publikacji są zapisane w osobnym pliku .bib. Pozwala to wielokrotnie wykorzystywać tę samą bazę.

10.1. Cytowanie

Opis zagadnienia można znaleźć
w literaturze~\cite{Doroszewski}.

10.2. Dołączenie bibliografii

\bibliographystyle{plain}
\bibliography{zbior}

10.3. Przykładowy wpis w pliku zbior.bib

@book{Lamport1994,
  author    = {Leslie Lamport},
  title     = {LaTeX: A Document Preparation System},
  year      = {1994},
  publisher = {Addison-Wesley}
}

10.4. Kolejność kompilacji

pdflatex wprowadzenie
bibtex wprowadzenie
pdflatex wprowadzenie
pdflatex wprowadzenie

Dlaczego cztery kroki? Pierwsza kompilacja zbiera cytowania, BibTeX tworzy listę literatury, a kolejne kompilacje uzupełniają numery cytowań, odwołania i strony.

10.5. Zmiana stylu

\bibliographystyle{alpha}
Klucz bibliograficzny, np. Lamport1994, musi być unikatowy. Brak zgodności między kluczem w \cite i pliku .bib prowadzi do nieoznaczonego cytowania.

11. Zadanie dodatkowe: circuitikz

Diagram jako kod. Obwód zapisany tekstowo jest skalowalny, łatwy do modyfikacji i może być wersjonowany razem z dokumentem.

11.1. Pakiet

\usepackage[
  smartlabels,
  european
]{circuitikz}

11.2. Obwód podstawowy

\begin{figure}[!htp]
\centering

\ctikzset{bipoles/length=1cm}
\ctikzset{inductors/scale=1,inductor=american}

\begin{circuitikz}[scale=1]
\draw
  (0,1) to[cspst=$u$] (1.5,1)
  (0,-1) -- (5,-1)
  (0,-1) to[battery,l_=$V_s$,invert] (0,1)
  (1.5,1) to[L,l=$L_1$,i<^=$I_1$] (1.5,-1)
  (1.5,1) to[C,l=$C_1$,v>={{$5V$}}] (3.5,1)
  (3.5,1) to[diode,l=$D_1$] (3.5,-1)
  (5,-1) to[R,l_=$R_1$] (5,1)
  (3.5,1) -- (5,1);
\end{circuitikz}

\caption{Obwód elektroniczny}
\label{fig:obwod}
\end{figure}

11.3. Rozszerzenie

Dodaj za rezystorem R1 diodę D2 skierowaną przeciwnie do D1 oraz kondensator C2 połączony szeregowo.

Współrzędne w nawiasach, np. (3.5,1), opisują położenie elementów w układzie kartezjańskim. Zmiana współrzędnych wpływa na geometrię całego obwodu.

13. Zakończenie ćwiczenia

Archiwizacja projektu. Dokument LaTeX składa się z wielu plików źródłowych. Należy zachować nie tylko PDF, ale również pliki .tex, .bib, kod źródłowy i grafiki.

  1. Sprawdź, czy dokument kompiluje się bez błędów.
  2. Otwórz wynikowy PDF i zweryfikuj podpisy, listingi, bibliografię oraz załącznik.
  3. Zarchiwizuj wszystkie pliki projektu.
  4. Po akceptacji prowadzącego usuń pliki ze stanowiska.
Nie usuwaj projektu przed sprawdzeniem, czy kopia zapasowa zawiera również pliki bibliografii, kodu źródłowego i grafiki.

14. Dodatkowe środowiska i sposoby pracy z LaTeX

Dobór środowiska do zadania. Ten sam dokument LaTeX może być rozwijany w przeglądarce, w lokalnym edytorze, z pomocą AI albo w kontrolowanym procesie opartym na Git i automatycznej kompilacji. Wybór zależy od wymagań projektu, liczby współautorów, potrzeby pracy offline i oczekiwanej powtarzalności procesu budowania dokumentu.

Overleaf: edycja i współpraca w środowisku chmurowym

Overleaf jest internetowym edytorem LaTeX oferującym edytor kodu i edytor wizualny, automatyczną kompilację, szablony oraz współpracę nad jednym projektem. Jest szczególnie przydatny podczas pracy zespołowej, konsultowania dokumentu z prowadzącym i szybkiego rozpoczęcia projektu bez lokalnej instalacji TeX.

Typowy tok pracy:

  1. Utwórz projekt albo wybierz szablon.
  2. Prześlij pliki .tex, .bib, grafiki i listingi.
  3. Wskaż główny plik dokumentu.
  4. Skompiluj projekt i przeanalizuj dziennik błędów.
  5. Udostępnij projekt współautorowi.

Materiały dydaktyczne: Overleaf Learn.

OpenAI Prism: środowisko LaTeX wspomagane przez AI

OpenAI Prism jest chmurową przestrzenią roboczą LaTeX do pisania naukowego, łączącą edycję, kompilację, współpracę i wsparcie sztucznej inteligencji uwzględniającą kontekst projektu. Może wspierać redakcję tekstu, analizę struktury dokumentu, pracę z równaniami, cytowaniami i literaturą.

Przykładowe zastosowania:

  • wyjaśnianie błędów kompilacji i proponowanie poprawek,
  • porządkowanie rozdziałów i sekcji,
  • redakcja stylu naukowego z zachowaniem składni LaTeX,
  • tworzenie tabel, równań i fragmentów kodu LaTeX,
  • analiza cytowań i bibliografii w kontekście dokumentu.

Informacje o środowisku: OpenAI Prism.

Edytor lokalny: VS Code, TeXstudio lub VIM

Praca lokalna polega na instalacji dystrybucji TeX, np. TeX Live albo MacTeX, oraz wybranego edytora. VS Code z rozszerzeniem LaTeX Workshop zapewnia podgląd PDF, diagnostykę i automatyczne budowanie. TeXstudio oferuje zintegrowany interfejs przeznaczony bezpośrednio do LaTeX, natomiast VIM pozwala pracować całkowicie z terminala.

Zalety:

  • pełna kontrola nad wersją kompilatora i pakietów,
  • praca bez dostępu do internetu,
  • łatwa integracja z lokalnymi skryptami, Pythonem, Git i narzędziami systemowymi,
  • możliwość obsługi dużych projektów i niestandardowych pakietów.

Ograniczenie: środowisko trzeba samodzielnie zainstalować, skonfigurować i aktualizować.

Repozytorium Git z automatyczną kompilacją

W tym modelu źródła dokumentu są przechowywane w repozytorium Git, a plik PDF jest budowany automatycznie przez system CI, np. GitHub Actions lub GitLab CI. Każda zmiana jest powiązana z commitem, a proces budowania może być wykonywany w identycznym, kontrolowanym środowisku.

Typowy proces:

  1. Autor edytuje dokument lokalnie na osobnej gałęzi.
  2. Zmiany są zatwierdzane i wysyłane do repozytorium.
  3. CI uruchamia latexmk lub odpowiednią sekwencję kompilacji.
  4. System zapisuje PDF jako artefakt i zgłasza błędy.
  5. Po przeglądzie zmian gałąź jest scalana z gałęzią główną.

Zaleta dydaktyczna: student uczy się jednocześnie LaTeX, kontroli wersji, przeglądu zmian i reprodukowalności procesu.
